# 🔐 КриптоКлюч × eTRN — полная карта работы

> Документ для всех: бэкендера, Android-разработчика, продакта, тестировщика.
> Объясняет КАК это работает с двух сторон одновременно — что видит юзер и что
> делается под капотом.

**Кто партнёр:** РосЭлТорг + СТ Крипт. Их продукт — **CKey SDK** (нативный Android/iOS) + сервер `key.roseltorg.ru` + сервис **Ордерс** для бэка.

---

```
   └────────────────────────────────────────────────────────────────────────┘
```

**Ключевая идея:**
- **Ключ подписи лежит на сервере КриптоПРО**, не у юзера на телефоне
- **Юзер на телефоне ПОДТВЕРЖДАЕТ** действия (биометрия / пароль)
- **Наш Android-app встраивает SDK** через который происходит подтверждение
- **Наш бэкенд через Ордерс API** заводит юзеров и отправляет документы на подпись

Это называется **«облачная подпись с мобильным подтверждением»**.

## 2. Большая картина — две основные истории

### История A: 🆕 Юзер первый раз ставит eTRN и выпускает КЭП

```
1️⃣  Скачал → 2️⃣ Зарегился → 3️⃣ Выпустил КЭП → 4️⃣ Готов подписывать
```

### История B: 📝 Юзер подписывает документ ЭТрН

```
1️⃣  Получил документ → 2️⃣ Открыл → 3️⃣ Тапнул «Подписать» → 4️⃣ Биометрия → ✅
```

Обе истории разберём по шагам с экранами и технической изнанкой.

### Resumé истории A — что должно быть на бэке и в приложении

| Что | Кто делает |
|---|---|
| Хранилище `applications` (заявки на выпуск) | КУБ-бэк |
| Хранилище `certificates` (готовые КЭП) | КУБ-бэк |
| `POST /sign/certificate/start` создаёт application в Ордерс | КУБ-бэк |
| `CKey.init()` в `Application.onCreate()` | Android |
| `UsersManager.createUser()` + `acceptAccountChanges()` | Android |
| `AuthMethodsManager.enableBiometric()` | Android |
| Polling статуса заявки | Android |
| `GET /certificates/me` в `Dashboard` | Android+бэк |

## 6. 🔄 Sequence-диаграммы

### Регистрация и выпуск КЭП (детально)

```mermaid
sequenceDiagram
    autonumber
    actor U as 👤 Юзер
    participant A as 📱 eTRN App
    participant K as 🔐 CKey SDK
    participant B as 🖥 КУБ-бэк
    participant O as 📋 Ордерс API
    participant S as ☁ key.roseltorg.ru
    participant M as 📧 SMTP
    participant F as 🏛 ФНС

    Note over U,F: Стадия 1: Авторизация в eTRN (своя)

    U->>A: Открыл приложение
    A->>K: CKey.init(rootCert)
    K-->>A: ✅ initialized
    A->>B: POST /auth/otp/send
    B-->>A: { verificationId }
    A-->>U: Введите код из SMS
    U->>A: Вводит код
    A->>B: POST /auth/otp/verify
    B-->>A: { JWT, user }

    Note over U,F: Стадия 2: Заявка на выпуск КЭП

    U->>A: Тапнул "Выпустить КЭП"
    A->>B: POST /sign/certificate/start (passport, snils)
    B->>O: createApplication()
    O->>S: ▶ inits application
    S-->>O: { applicationId, ckeyAppId }
    O-->>B: { applicationId, ckeyAppId }
    B-->>A: { applicationId, ckeyAppId }

    Note over U,F: Стадия 3: Идентификация (ЕСИА)

    A->>U: Открой Госуслуги
    U->>U: Подтвердил через ЕСИА
    U->>A: Вернулся
    A->>B: GET /sign/certificate/status

    Note over U,F: Стадия 4: Привязка устройства к КриптоКлюч

    U->>A: Тапнул "Привязать устройство"
    A->>K: UsersManager.createUser(email, ckeyAppId)
    K->>S: POST register-pre
    S->>M: send OTP to email
    M-->>U: 📧 Код: 482917
    K-->>A: { userId, status: AWAITING_OTP }

    U->>A: Ввёл код
    A->>K: UsersManager.acceptAccountChanges(otp)
    K->>S: POST confirm
    S-->>K: ✅ activated
    K-->>A: success

    A->>K: enableBiometric(userId)
    K-->>A: ✅ biometric enrolled

    Note over U,F: Стадия 5: Выпуск самого сертификата

    A->>B: POST /sign/certificate/issue
    B->>O: requestCertIssue()
    O->>S: подписать заявление
    S->>F: запрос сертификата
    F-->>S: сертификат
    S-->>O: { certificate }
    O-->>B: { certificate }
    B->>B: save certificate
    B-->>A: ✅ done

    A-->>U: 🎉 Готово! Можете подписывать
```

### Подписание ЭТрН

```mermaid
sequenceDiagram
    autonumber
    actor U as 👤 Юзер
    participant A as 📱 eTRN App
    participant K as 🔐 CKey SDK
    participant B as 🖥 КУБ-бэк
    participant O as 📋 Ордерс API
    participant S as ☁ key.roseltorg.ru
    participant E as 📨 СБИС/Диадок

    U->>A: Открыл документ ЭТрН
    A->>B: GET /mcd/find-for-signing
    B-->>A: { mcd, certificate }
    A-->>U: 🟢 "Подписание по МЧД №..."

    U->>A: Тап "Подписать"
    A->>B: POST /documents/{id}/sign/init
    B->>O: createOperation(documentXml, mcdId)
    O->>S: ▶ создать operation
    S-->>O: { operationId }
    O-->>B: { operationId }
    B-->>A: { operationId }

    A->>K: OperationsManager.getOperation(operationId)
    K->>S: GET operation
    S-->>K: { docs, info, signFields }
    K-->>A: Operation

    Note over A: SDK сам показывает экран подтверждения

    K->>U: 🔐 Экран SDK: "Вы подписываете ЭТрН..."
    U->>K: 👆 Тап "Подписать"
    K->>U: 🔐 Запрос биометрии
    U->>K: 👆 Отпечаток
    K->>S: confirmOperation(authData)
    S->>S: расшифровать ключ + подписать
    S-->>K: { signature }
    K-->>A: { signature }

    A->>B: POST /documents/{id}/sign/submit { signature, operationId }
    B->>E: send signed envelope
    E-->>B: ✅ accepted
    B->>B: document.status = SIGNED
    B-->>A: ✅ done

    A-->>U: ✅ Документ подписан!
```

## 8. ❌ Что НЕ работает / типичные траблы

| Проблема | Почему | Решение |
|---|---|---|
| `ERROR_NOT_INITIALIZATED` при createUser | Забыли `CKey.init()` в `Application` | Init **в `onCreate` Application-класса**, не в Activity |
| QR требуется при createUser | applicationId настроен «QR-only» админом УЦ | Попросить Никиту переключить на «open» режим |
| После подписи документ не уходит в СБИС | Бэк не подключён к адаптеру СБИС | Реализовать adapter в `edo-operators` модуле |
| Биометрия не работает | Юзер сменил палец / не настроена | SDK fallback на пароль через `AuthData.Password()` |
| Сертификат истёк | Срок 12 мес | Cron на бэке + push «обновите КЭП за N дней» |
| Подпись отклонена оператором ЭДО | МЧД невалидна / ИНН не совпадает | Использовать `findMcdForSigning()` ДО `sign/init` |
| dev-стенд недоступен | VPN-проблемы (сейчас затык!) | Догнать Романа Артюха |
