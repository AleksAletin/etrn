# Примеры XML-МЧД для тестов

## `mcd-example.xml`

Пример машиночитаемой доверенности в формате ФНС.

**Формат:** Приказ ФНС России от 19.09.2023 № ЕД-7-26/648@ (версия 003).

### Что в файле

- **Доверитель:** ООО «ТрансЛогистик» (ИНН 7712345678, КПП 771201001, ОГРН 1027700123456)
- **Руководитель (подписант):** Смирнов Алексей Николаевич
- **Доверенное лицо:** Иванов Сергей Петрович (ИНН 123456789012, физлицо)
- **Срок:** с 2026-01-15 по 2027-01-15
- **Полномочия:**
  - `02.08` — Подписание ЭТрН
  - `02.09` — Подписание заказ-нарядов (ЭЗЗ)
  - `06.01` — Просмотр документов

### Что должен распарсить бэк из этого файла

Согласно [`docs/mcd-spec-v2.md`](../mcd-spec-v2.md), endpoint `POST /api/v1/mcd/parse` должен вернуть:

```json
{
  "parsed": {
    "number": "МЧД-2026-00456",
    "principal": {
      "companyName": "ООО «ТрансЛогистик»",
      "inn": "7712345678"
    },
    "trustedPerson": "Иванов Сергей Петрович",
    "validUntil": "2027-01-15",
    "powers": [
      { "code": "02.08", "name": "Подписание электронных транспортных накладных (ЭТрН)" },
      { "code": "02.09", "name": "Подписание заказ-нарядов (ЭЗЗ)" },
      { "code": "06.01", "name": "Просмотр электронных документов" }
    ]
  }
}
```

### Где брать реальные эталонные XSD-схемы

ФНС публикует актуальные схемы на сайте:
- https://www.nalog.gov.ru/rn77/about_fts/docs/14846210/ (раздел «Форматы электронной доверенности»)
- Архив со схемами: `dover_{версия}.zip` — содержит `.xsd` и примеры заполнения

**Важно:** формат регулярно обновляется. Перед продакшеном — сверить версию с актуальной на сайте ФНС.

### Как получить образец от реального оператора

Если нужны «живые» примеры — можно запросить у операторов ЭДО (СБИС, Диадок, Контур), они публикуют образцы на страницах для разработчиков. Либо выпустить свою тестовую МЧД в личном кабинете ФНС и использовать её как эталон.

### Ограничения этого примера

- Подпись (XMLDSig) приведена в упрощённом виде (блок `<Подписант>` вместо реального `<Signature>`). В проде нужна полноценная CAdES/XAdES подпись.
- Некоторые справочные поля (коды регионов, КПП, ОКВЭД) могут не соответствовать реальным записям в ЕГРЮЛ — это демо-данные.
